# Knowledge Vault

`/deep` persists long-lived knowledge — ubiquitous-language glossary
terms, architecture decision records, and curated discovery findings —
into an Obsidian-flavored vault that survives across sessions and
projects.

The vault is optional. When unavailable, every artifact still lands in
the project memory directory (`~/.claude/projects/<slug>/memory/`) so
the workflow does not break.

## Resolution order

`scripts/lib/vault.py::resolve_vault_path()` returns a `VaultStatus`
object with one of three sources:

| `source` | `exists` | What `/deep` does |
|---|---|---|
| `env` | `true` | Use `$DEEP_OBSIDIAN_VAULT` as the vault root. |
| `env` | `false` | Warn that the env path is missing, fall back to the default and re-resolve. |
| `default` | `true` | Use `~/Obsidian/deep-plan/`. |
| `default` | `false` | Same path string but the directory does not exist; treated as `none`. |
| `none` | `false` | Prompt the user once via `AskUserQuestion`. |

The first-run prompt is **the only** time `/deep` creates a vault.
"Yes" runs `ensure_vault_skeleton` at the default path; "specify path"
runs it at the user-supplied path; "no" sets `vault_available = false`
for the session and skips vault writes entirely. The "no" answer is
remembered for the session — `/deep` will not re-ask the same prompt
twice.

## Layout

```
<vault>/
├── .obsidian/                  # minimal config so the app opens cleanly
├── README.md                   # generated by ensure_vault_skeleton
├── _index.md                   # top-level Map of Content
├── projects/<slug>/            # per-project notes
├── glossary/<slug>/            # per-project ubiquitous-language terms
├── glossary/_global/           # cross-project promotions
├── adrs/<slug>/                # architecture decision records
└── findings/<slug>/            # curated discovery output
```

`<slug>` is `vault.slugify_project(project_path)` — the lowercased
basename of the project root with non-alphanumeric characters
collapsed to single hyphens.

## What lands here

A `vault-curator` subagent (see `agents/vault-curator.md`) decides per
artifact at the end of every `/deep` mode. Default routing:

| Artifact | Default destination | Why |
|---|---|---|
| Ubiquitous-language glossary terms | `glossary/<slug>/` | Stable across sessions. |
| ADRs from architecture audit | `adrs/<slug>/` | Decision plus rationale. |
| Architecture-audit findings (when user accepted a deepening) | `findings/<slug>/architecture-audit-<date>.md` | Records why the deepening was chosen. |
| `findings/skills-considered.md` | local only | Useful for the run, not future ones. |
| `claude-plan.md` | local only | Code becomes the source of truth. |
| `claude-interview.md` | local only | Save only flagged Q&As to `projects/<slug>/notes.md`. |
| `.deepstate/state.json` | local only | Pure session bookkeeping. |
| `impl-summary.md` | local only | Append public interfaces to `projects/<slug>/interfaces.md`. |

Cross-project hits (a term or ADR seen in two or more projects) get
promoted to the `_global/` subdirectory under the appropriate top-level
folder. The curator tags them with both project slugs.

## Manual override

`--no-vault` on any `/deep` invocation forces vault writes off for the
session. Useful when working on sensitive code where you do not want
artifacts replicated to a notes app.

`scripts/lib/vault.py` is stdlib-only; its functions are deterministic
and safe to call from tests or other scripts.
